#### 3.3.3 Stacking集成（v18 → v19）

**v18问题版本**（Ridge元学习器）:
```python
# 问题：仅在单个验证集上拟合元学习器
meta_model = Ridge()
meta_model.fit(val_preds, y_val)  # val_preds形状 (7000, 3)
# 结果：权重异常 [LGB=-0.225, XGB=1.039, CAT=0.323]
```

**v19修复版本**（OOF Stacking + 非负约束）:
```python
# 5折OOF，构建完整元特征矩阵 (N, 3)
for fold in 5_folds:
    train_fold = 数据[:fold_start] + 数据[fold_end:]
    val_fold = 数据[fold_start:fold_end]
    
    lgb_model.fit(train_fold)
    xgb_model.fit(train_fold)
    cat_model.fit(train_fold)
    
    oof_preds[val_fold] = [lgb_pred, xgb_pred, cat_pred]

# 非负约束优化权重（scipy SLSQP）
def objective(weights):
    weighted_pred = oof_preds @ weights
    return mean_absolute_error(y_true, weighted_pred)

constraints = [
    {'type': 'eq', 'fun': lambda w: np.sum(w) - 1},  # 权重和=1
    {'type': 'ineq', 'fun': lambda w: w}           # 所有权重≥0
]
result = minimize(objective, x0=[1/3, 1/3, 1/3], constraints=constraints, method='SLSQP')
```

**v19结果**:
```

非负约束最优权重 [LGB, XGB, CAT]: [0.1455  0.      0.8545]
OOF加权融合MAE: 0.4274

最终模型验证集MAE:
  LGB: 0.3152
  XGB: 0.3227
  CAT: 0.3155
  加权融合: 0.3147  ← 最佳
```

## 四、充放电策略

### 4.1 穷举优化（所有版本通用）

```python
# 遍历所有合法的(tc, td)组合
best_profit = 0
for tc in range(0, 81):           # 充电开始时间（0-80）
    charge_cost = sum(prices[tc:tc+8]) * 1000
    for td in range(tc+8, 89):    # 放电开始时间（tc+8 ~ 88）
        discharge_revenue = sum(prices[td:td+8]) * 1000
        profit = discharge_revenue - charge_cost
        if profit > best_profit:
            best_profit = profit
            best_tc, best_td = tc, td

# 生成策略
power[best_tc:best_tc+8] = -1000   # 充电
power[best_td:best_td+8] = +1000   # 放电
```

**问题**: sklearn接口不支持early_stopping，需要手动实现

```python
best_score = float('inf')

for i in range(max_iterations):
    model.set_params(n_estimators=(i+1)*10)
    model.fit(X_train, y_train)
```

## 七、关键反思：为什么V19得分反而更低？

### 7.1 比赛得分对比

| 模型 | 本地验证MAE | 本地验证RMSE | **比赛得分** |
|------|-------------|--------------|--------------|
| **LGBM_baseline** | 0.2988 | 0.3978 | **5403.46** |
| **v19_fixed_ensemble** | 0.3147 | 0.5037 | **4290.73** |

> ⚠️ **重大教训**: V19虽然本地MAE仅比Baseline高0.016，但比赛得分低了**1113分**！

### 7.2 根因分析

#### 1. RMSE vs MAE - 关键指标

| 模型 | MAE | RMSE | RMSE/MAE比率 |
|------|-----|------|--------------|
| Baseline | 0.2988 | 0.3978 | **1.33** |
| V19 | 0.3147 | 0.5037 | **1.60** |

**发现**: V19的RMSE/MAE比率远高于Baseline，说明存在**极端预测误差**。

#### 2. 极端误差的代价

```
V19电价预测统计:
- 均值: 0.5663 (Baseline: 0.5788)
- 标准差: 0.5333 (Baseline: 0.5408)
- 最大值: 2.0112 (Baseline: 1.8266) ← 极端高估！
- 最小值: -0.0321 (Baseline: -0.0426)
```

#### 3. 策略决策差异

**Power策略不同的记录: 674条（占12%）**

| 差异类型 | 记录数 | 影响 |
|----------|--------|------|
| V19充电但Baseline不充 | 186 | 可能高价充电 → 损失 |
| V19放电但Baseline不放 | 151 | 可能低价放电 → 损失 |
| Baseline充电但V19不充 | 186 | V19错过低价充电机会 |
| Baseline放电但V19不放 | 151 | V19错过高价放电机会 |

#### 4. 充放电时机错误分析

```
V19错误充电的平均电价: ¥0.0533/kWh
Baseline正确充电的平均电价: ¥0.0719/kWh

→ V19在更高价时段充电，损失了 ¥0.0186/kWh × 8小时 × 1000kW = ¥149/天
→ 约59天测试期 → 累计损失约 ¥8,791
```

### 7.3 核心教训

#### ❌ 错误认知

> "MAE降低了0.016，所以V19一定更好"

这是**致命错误**！因为：

1. **MAE只看平均误差，不看分布**
   - MAE=0.31 可能由以下两种情况产生：
     - A: 所有样本误差都是±0.31（均匀分布）
     - B: 90%样本误差0.1，10%样本误差3.1（极端分布）
   
2. **充放电策略对极端值极度敏感**
   - 充电选在低价时段，放电选在高价时段
   - 一旦预测错误（尤其高估低价），策略会选错时机
   - 一次错误决策的损失可能抵消数十次正确决策的收益

3. **RMSE才是评估策略适用性的关键指标**
   - RMSE对大误差惩罚更重
   - RMSE/MAE比率可以衡量预测的"稳定性"

#### ✅ 正确认知

> "在充放电优化场景中，应该同时优化MAE和RMSE"

**改进方向**:

1. **使用Huber Loss替代MAE**
   ```python
   # Huber Loss = MAE + 对大误差的二次惩罚
   # 兼顾鲁棒性和对极端值的敏感性
   ```

2. **添加RMSE作为评估指标**
   ```python
   from sklearn.metrics import mean_squared_error
   rmse = np.sqrt(mean_squared_error(y_true, y_pred))
   
   # 约束条件
   if rmse > threshold:  # 拒绝预测不稳定的模型
       reject_model()
   ```

3. **双目标优化**
   ```python
   # 目标1: 最小化MAE
   # 目标2: 最小化RMSE
   # 选择帕累托最优解
   ```

### 8.3 关键发现

1. **advance_LGBM_baseline MAE最低(0.2988)**：但未在竞赛中提交验证
2. **v19 RMSE较高(0.5037)**：说明预测存在少数大误差
   - MAE=0.3147 < RMSE=0.5037 → 存在异常大误差
   - 对电价预测来说，大误差会导致交易策略失效
   - 建议：检查v19的预测分布，是否存在极端预测值
3. **v18虽然MAE较高(0.3340)**：但预测偏差方向有利于充放电策略，收益反而更高
4. **RMSE与MAE的差异**：反映模型对异常值的敏感性
   - RMSE ≈ MAE → 误差分布均匀
   - RMSE > MAE → 存在少数大误差（v19的情况）

### 9.2 非负约束权重优化

```python
# v19_fixed_ensemble.py
from scipy.optimize import minimize

def objective(weights):
    pred = np.dot(oof_preds, weights)
    return mean_absolute_error(y_true, pred)

# 约束：权重≥0，权重和=1
constraints = [
    {'type': 'eq', 'fun': lambda w: np.sum(w) - 1},
    {'type': 'ineq', 'fun': lambda w: w}
]
result = minimize(objective, x0=[1/3, 1/3, 1/3], 
               constraints=constraints, method='SLSQP')
optimal_weights = result.x  # e.g., [0.1455, 0.0, 0.8545]
```
